# Essential Terraform Commands Interview Questions

## Most Frequently Asked Interview Questions

### Question 1

**Question**

What is the purpose of the `terraform init` command?

**Answer**

`terraform init` initializes a Terraform working directory by:

- Downloading required providers
- Installing modules
- Configuring the backend
- Preparing the project for execution

Command:

```bash
terraform init
```

**Explanation**

`terraform init` is the first command executed in every Terraform project. Without initialization, Terraform cannot communicate with cloud providers or manage infrastructure.

**Used in Production**

- New Terraform projects
- After changing providers
- After modifying backend configuration
- After adding modules

**Common Mistake**

Skipping `terraform init` after cloning a repository or changing backend/provider configurations.

---

### Question 2

**Question**

What does the `terraform validate` command do?

**Answer**

`terraform validate` checks whether the Terraform configuration is syntactically correct and internally consistent.

Command:

```bash
terraform validate
```

**Explanation**

It validates:

- HCL syntax
- Resource references
- Variable definitions
- Configuration structure

It does **not** contact the cloud provider or create resources.

**Used in Production**

- CI/CD validation stage
- Pre-deployment checks
- Code reviews

**Common Mistake**

Assuming `terraform validate` checks infrastructure changes. It only validates configuration.

---

### Question 3

**Question**

Why is the `terraform fmt` command important?

**Answer**

`terraform fmt` automatically formats Terraform configuration files according to Terraform's standard style.

Command:

```bash
terraform fmt
```

**Explanation**

It improves:

- Code readability
- Team consistency
- Maintainability

It only changes formatting and does not modify infrastructure.

**Used in Production**

- Before committing code
- CI/CD formatting checks
- Team collaboration

**Common Mistake**

Ignoring formatting standards, leading to inconsistent code across projects.

---

### Question 4

**Question**

What is the purpose of the `terraform plan` command?

**Answer**

`terraform plan` previews infrastructure changes before they are applied.

Command:

```bash
terraform plan
```

The output shows:

- Resources to create
- Resources to modify
- Resources to destroy

**Explanation**

Terraform compares the desired configuration with the current state and generates an execution plan.

**Used in Production**

- Change reviews
- Approval workflows
- CI/CD pipelines

**Common Mistake**

Skipping the Plan stage and applying changes directly.

---

### Question 5

**Question**

What does the `terraform apply` command do?

**Answer**

`terraform apply` executes the changes generated by the execution plan and provisions or updates infrastructure.

Command:

```bash
terraform apply
```

**Explanation**

Terraform communicates with the cloud provider and makes the infrastructure match the desired configuration.

**Used in Production**

- Infrastructure provisioning
- Infrastructure updates
- Automated deployments

**Common Mistake**

Running `terraform apply` without reviewing the execution plan.

---

### Question 6

**Question**

When should you use the `terraform destroy` command?

**Answer**

`terraform destroy` removes all infrastructure managed by the current Terraform configuration.

Command:

```bash
terraform destroy
```

**Explanation**

Terraform deletes resources in dependency order to avoid failures.

**Used in Production**

- Temporary environments
- Lab environments
- Development cleanup
- Disaster recovery testing

**Common Mistake**

Accidentally running `terraform destroy` against production infrastructure.

---

### Question 7

**Question**

What is the purpose of the `terraform output` command?

**Answer**

`terraform output` displays output values defined in the Terraform configuration.

Command:

```bash
terraform output
```

Example outputs:

- VM Public IP
- Storage Account Name
- Resource IDs
- Connection strings

**Explanation**

Outputs make important infrastructure information available after deployment.

**Used in Production**

- CI/CD pipelines
- Automation scripts
- Integration with other tools

**Common Mistake**

Forgetting to define outputs for commonly required resource information.

---

### Question 8

**Question**

What does the `terraform show` command display?

**Answer**

`terraform show` displays detailed information about the current Terraform state or execution plan.

Command:

```bash
terraform show
```

**Explanation**

It provides:

- Resource details
- Current state
- Attribute values
- Planned infrastructure changes

**Used in Production**

- Troubleshooting
- State verification
- Infrastructure audits

---

### Question 9

**Question**

What is the purpose of the `terraform state` command?

**Answer**

`terraform state` is used to inspect and manage the Terraform state file.

Common commands include:

```bash
terraform state list
terraform state show
terraform state mv
terraform state rm
```

**Explanation**

The state command helps synchronize Terraform's knowledge of infrastructure with the actual cloud resources.

**Used in Production**

- State management
- Resource migration
- Troubleshooting
- Refactoring Terraform code

**Common Mistake**

Editing the state file manually instead of using Terraform state commands.

---

### Question 10

**Question**

What is the recommended order of Terraform commands during a deployment?

**Answer**

A typical workflow is:

```text
terraform init
        ↓
terraform fmt
        ↓
terraform validate
        ↓
terraform plan
        ↓
terraform apply
```

**Explanation**

This sequence ensures that the project is initialized, formatted, validated, reviewed, and then safely deployed.

**Used in Production**

Nearly every enterprise Terraform pipeline.

---

### Question 11

**Question**

Which Terraform commands are most commonly used in CI/CD pipelines?

**Answer**

The most common commands are:

- `terraform init`
- `terraform fmt`
- `terraform validate`
- `terraform plan`
- Manual Approval (optional)
- `terraform apply`

Some pipelines also use:

- `terraform output`
- `terraform show`

**Explanation**

These commands automate infrastructure deployment while ensuring validation and change review.

**Used in Production**

- Azure DevOps
- Jenkins
- GitHub Actions
- GitLab CI/CD

---

### Question 12

**Question**

How would you troubleshoot a failed Terraform deployment using Terraform commands?

**Answer**

A structured troubleshooting approach is:

1. Run `terraform validate`
2. Review `terraform plan`
3. Inspect resources using `terraform show`
4. Check outputs using `terraform output`
5. Inspect the state using `terraform state list` or `terraform state show`
6. Fix the issue and rerun `terraform apply`

**Explanation**

Each command provides different diagnostic information that helps identify configuration, state, or deployment issues.

**Used in Production**

Daily troubleshooting of Infrastructure as Code deployments.

# Scenario-Based Interview Questions

### Scenario 1

**Question**

A teammate clones a Terraform repository and immediately runs `terraform plan`, but receives provider-related errors. What is the likely issue?

**Answer**

They did not run `terraform init` first.

**Explanation**

`terraform init` downloads providers, modules, and configures the backend. Without initialization, Terraform cannot execute plans.

---

### Scenario 2

**Question**

Your CI/CD pipeline fails because of incorrect Terraform formatting. Which command should be included before validation?

**Answer**

Run:

```bash
terraform fmt
```

**Explanation**

Formatting ensures consistent code style and helps prevent pipeline failures caused by formatting checks.

---

### Scenario 3

**Question**

You modified several Terraform files and want to verify the configuration before making any infrastructure changes. Which command should you use?

**Answer**

Run:

```bash
terraform validate
```

**Explanation**

This checks for syntax and configuration errors without contacting the cloud provider.

---

### Scenario 4

**Question**

Your manager asks what infrastructure changes will occur before approving a deployment. Which command provides this information?

**Answer**

Run:

```bash
terraform plan
```

**Explanation**

The execution plan previews all proposed infrastructure changes without applying them.

---

### Scenario 5

**Question**

A developer accidentally executes `terraform apply` directly in production without reviewing the changes. How can this be prevented?

**Answer**

Implement a CI/CD workflow that requires:

1. `terraform plan`
2. Manual approval
3. `terraform apply`

**Explanation**

Separating the Plan and Apply stages reduces the risk of unintended production changes.

---

### Scenario 6

**Question**

After deployment, the application team needs the Virtual Machine's public IP address. Which Terraform command would you use?

**Answer**

Run:

```bash
terraform output
```

**Explanation**

Output values expose important infrastructure information such as IP addresses, hostnames, and resource IDs.

---

### Scenario 7

**Question**

Your Terraform deployment completed successfully, but you want to inspect the current resource details stored in the state. Which command would you use?

**Answer**

Run:

```bash
terraform show
```

**Explanation**

`terraform show` displays detailed information about the current state, including resource attributes and configuration.

---

### Scenario 8

**Question**

Your infrastructure team suspects that the Terraform state is out of sync with Azure resources. Which command should you use first?

**Answer**

Run:

```bash
terraform state list
```

or

```bash
terraform state show
```

**Explanation**

These commands help inspect the current state and identify discrepancies between Terraform and the actual infrastructure.

---

### Scenario 9

**Question**

A development environment is no longer needed. How would you safely remove all infrastructure created by Terraform?

**Answer**

Run:

```bash
terraform destroy
```

After reviewing the resources to be deleted.

**Explanation**

`terraform destroy` removes all managed resources while respecting dependency order.

---

**Question**

Your team wants a standardized Terraform deployment workflow for all projects. What sequence of commands would you recommend?

**Answer**

A recommended workflow is:

```text
terraform init
        ↓
terraform fmt
        ↓
terraform validate
        ↓
terraform plan
        ↓
Manual Approval
        ↓

terraform output
```
